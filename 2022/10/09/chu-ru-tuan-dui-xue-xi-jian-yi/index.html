1. 学习计划建议（基础）

  • 基础知识：仔细学习每一部分内容，熟练掌握原理与思想，推导公式，（1~2个月）
    1. 机器学习资料：斯坦福机器学习网易公开课
    2. 机器学习 周志华 清华大学出版社
    3. Pattern Recognition and Machine Learning
  • Linux：没必要花大量时间看这方面的书或者视频，日常熟练使用即可（1周）
    1. 鸟哥的Linux私房菜
  • Python学习资料：可与机器学习同时学习，建议在linux下学习（一个月）
    1. 《Head first python》 ：（简单易入门，但是内容较浅显）
    2. 《机器学习实战》：用python实现机器学习算法
    3. 小甲鱼《零基础学习python》:
    4. Python基础教程（第二版• 修订版）【挪】Magnus Lie Hetland 著 （全面深入）
  • 深度学习资料：入门，掌握基本框架
    Neural Network and Deep Learning: （从神经网络到深度学习的一个入门教程）
    http://neuralnetworksanddeeplearning.com/index.html
    Ufldl教程：
    英文：http://ufldl.stanford.edu/wiki/index.php/UFLDL_Tutorial
    中文：http://ufldl.stanford.edu/wiki/index.php/UFLDL%E6%95%99%E7%A8%8B
  • 深度学习框架： 不局限于这些框架，挑选一个上手、熟练使用即可
    PyTorch教程：主页 - PyTorch中文文档 (pytorch-cn.readthedocs.io)
    Tensorflow教程：https://www.tensorflow.org/
    Theano教程：http://www.deeplearning.net/tutorial/
    Caffe教程：http://caffe.berkeleyvision.org/tutorial/
    MxNet教程：http://mxnet-bing.readthedocs.io/en/latest/
  • 深度学习目标检测算法：【熟练掌握分类任务的前提下】全面收录目标检测算法，学习深度学习目标检测的发展，认真阅读各经典算法论文，掌握其主要思想，可以将其公布的代码运用到自己感兴趣的领域 。
    https://handong1587.github.io/deep_learning/2015/10/09/object-detection.html

2. 前期的一些通用建议

加入TAPD

进行平时进度的管理和资料的整理，邀请函在最下方

  1. 平时整理的资料文档要及时上传到项目中的文档中，每人自己建立一个文件夹，及时整理手头资料。
  2. 每周日常进度安排可在看板中自行添加
  3. 等有代码复现后在TAPD的项目看板中贴上Github仓库链接
  4. 可以在WIKI中记录分享一些踩过的坑，知识共享

2.0. 科学上网

  • 必不可少，必不可少，必不可少，没这个别的无从谈起。
  • 推荐：iGuge - iGG谷歌学术助手，有免费线路，也可一个月15块钱，氪金后速度快很多。其他VPN都可。

    2.1. 文献查找

整体的思路：知乎或优质Blog整体介绍（快速入门）–>相关领域综述（中英都可）–>具体方法精读（英）–>找代码实现

  • 国内数据库：中国知网、万方数据等，可用来查找最最最基本的综述、硕博士论文，仅供参考即可，不做重点。

  • 国外数据库：IEEE查询（较常用）、 EI会议查询、SCI期刊查询

    • （注：国内数据库接入校园网可直接下载文献，国外数据库需要用学号密码登录进行机构认证方可下载）

  • Google 学术搜索：直接搜索英文关键词，部分文章pdf可以直接下载，少数可利用Sci-Hub通过DOI号下载。

    • 注：谷歌学术最常用，最重要，也常用于参考文献格式的导出（点文献下的引号“”即可）。

  • 除此之外，其他的一些小众神器：

    • 这个网站可以在找文献的同时找代码，并有某个领域的benchmark：The latest in Machine Learning | Papers With Code
    • 一款在线论文阅读笔记神器：ReadPaper，两亿篇免费文章，还有牛逼实用的阅读工具

  • 最后，一定要找牛逼的文献来读，用最新SCI期刊查询及投稿分析系统（2020-2021年） - LetPub来查询你下的文献期刊水平如何。

2.2. 代码查找

  • GitHub: Where the world builds software · GitHub 众所周知，还是以这个为主，自己的代码一定要建个仓库，经常push自己的代码留下修改历史是个好习惯
    • 小技巧：在一个仓库地址的github后面加个1s，可以使用VsCode预览代码，很方便
  • The latest in Machine Learning | Papers With Code 有些文献可以在这里找到代码，很方便
  • Gitee - 基于 Git 的代码托管和研发协作平台

2.3. 一些必备的工具

  • Ubuntu：没有的可以折腾个双系统，当然用Win也可，Linux省去很多麻烦。
  • Git：程序员必备的基础之一，注意代码的版本控制，经常PUSH，防止代码丢失，相关教程B站很多，不会的必须掌握下。
  • Markdown语言：也是程序员必备的基础之一。用来写各种东西，平时随手记的东西尽量用markdown，专注内容，无需排版，编辑器推荐Typora — a markdown editor, markdown reader.。
  • Visio：画图的工具，推荐使用。
  • MathType：所有公式尽量都用MathType来写，很多破解版可用。
  • 文献管理工具：例如Mendeley（免费），还有很多软件可用。
  • XMind：具体不用详细介绍了，可以用于整理搜集的参考资料、做脑图。
  • 知云文献翻译：不错的文献阅读工具，不过最好还是打印出来看原版。

2.4. 其他的一些注意事项

  1. 重要：遇到问题先Google，如解决不了再百度，最后实在无法解决再向学长学姐咨询，一定要有解决问题的能力
  2. 注意文档文件命名：至少包含：修改日期、修改人、版本
  3. word格式一般为：小四、宋体（中）、Times New Roman（英）、1.5倍行距
  4. 及时整理文档资料上传TAPD，注意整理
  5. 代码必须放至github仓库，用git进行版本控制，保存修改历史
  6. 代码注释良好，有头部信息，下面给出一个模板例子：
